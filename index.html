JavaScript is a single-threaded synchronous programming language.The event loop is the secret by which JavaScript gives us an illusion of being multithreaded even though it is single-threaded. When js engine get data, the javascript first divide his work in stack and queue. Stack work is synchronous work. So it's normally execute his code in order. If stack get any data to executed which have to wait such as fetch, call back function, then it send those data to Queue. the queue keep that data till stack section being empty. So when it's confirmed the stack is cleared and theres more data need to be executed then the queue move those data in stack section and it's being executed. The whole process done by event loop. so the event loop normally checks weather stack is cleard or not. if it is clears then it send Queue data into stack otherwise it'll wait and keep cheking. That's how event loop work in js engine.
